## איך מגדירים Render.com

### שלב 1: הכנת הפרויקט

צריך להוסיף כמה קבצים:

1. **`render.yaml`** - קובץ הגדרות ל-Render
2. **`.dockerignore`** (אופציונלי) - אם משתמשים ב-Docker
3. עדכון `package.json` - להוסיף script `start`

### שלב 2: יצירת חשבון ב-Render

1. היכנס ל-https://render.com
2. הירשם עם GitHub (מומלץ) או אימייל
3. חיבור ה-repository שלך

### שלב 3: יצירת Web Service

1. לחץ על "New +" → "Web Service"
2. בחר את ה-repository שלך
3. הגדר:
   - **Name**: `toledano-edtech-server`
   - **Environment**: `Node`
   - **Build Command**: `cd server && npm install`
   - **Start Command**: `cd server && npm start`
   - **Plan**: `Free`

### שלב 4: הגדרת משתני סביבה

ב-Render, הוסף Environment Variables:
- `FRONTEND_URL` = כתובת האתר שלך (למשל: `https://your-app.onrender.com`)
- `PORT` = `10000` (Render משתמש בפורט זה)

### שלב 5: עדכון ה-Frontend

עדכן את `components/FileUpload.tsx`:
```typescript
const API_BASE_URL = import.meta.env.VITE_API_URL || 'https://your-server.onrender.com';
```

## מגבלות Render.com (חינם)

1. **Spin-down**: השרת נרדם אחרי 15 דקות ללא פעילות
   - **פתרון**: אפשר להשתמש ב-UptimeRobot (חינם) כדי לשמור על השרת ער

2. **קבצים זמניים**: נמחקים בעת restart
   - **פתרון**: זה בסדר כי הקבצים זמניים בכל מקרה

3. **זמן התחלה**: עד דקה להתעוררות
   - **פתרון**: הוסף loading state ב-frontend
